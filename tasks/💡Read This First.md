
# **Project Workflow**

In this project, you will gain hands-on experience using a GitHub project board to manage your tasks. Below are the detailed steps you need to follow.

## **Label Meanings**
Issue with the label **"MVP"(minimum viable product)** is a mandatory feature.

Labels without "MVP" indicate **bonus** features.

## **Step 1: Understanding Issues and User Stories**

Each task you'll work on is represented by an **Issue** on the GitHub project board. The issues are written as **User Stories**.

A **User Story** is a short, simple description of a task or requirement from the perspective of an end-user. It typically follows this structure:

> **As a [user type], I want [something], so that [benefit].**

You don't need to master this concept right now—just know that it's a way of describing what needs to be done. During your project, you'll also find **Acceptance Criteria** listed in each issue. These criteria tell you exactly what needs to happen for the issue to be considered complete.

## **Step 2: Assigning and Starting Your Task**

1. **Decide** which team member will work on each issue. Assign the issue to the selected person on the GitHub project board.  
2. **Only one issue per person** should be in the "In Progress" column at a time.
3. Carefully read the **User Story** and **Acceptance Criteria** for the assigned issue.
4. **Create a new branch** in GitHub for this issue. Use a descriptive name for the branch.
   - Example: If the issue is about creating a login page, you could name the branch `feature/login-page`.
5. Link the branch to the issue.
6. **Move the issue** to the "In Progress" column of the GitHub board.

## **Step 3: Working on the Task**

- Work on the task in your newly created branch.
- Ensure that all the **Acceptance Criteria** are met before considering the task complete.

## **Step 4: Collaborating and Submitting Your Work**

1. Once you finish the task, **create a Pull Request (PR)** to merge your branch into the main branch.
2. Ask your team members to **review your PR** and provide feedback.
3. Review the code for any errors and confirm that all **Acceptance Criteria** are met.
4. Once approved, **merge the PR** into the main branch.

## **Step 5: Completing the Task**

1. Once the PR is merged, **move the issue** to the "Done" column on the GitHub board.
2. Repeat these steps for the next issue on the board.

## **Tips for Success**

- **Commit frequently:** Save your progress regularly by committing your code changes with descriptive messages.
- **Ask questions:** If anything is unclear, reach out to your instructor or peers for guidance.
- **Review each other's work:** When team members are working on different branches, it's essential to review each other's PRs to maintain code quality.
- **Follow GitHub best practices:** Use meaningful branch names, commit messages, and follow coding standards.
